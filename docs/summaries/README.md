# Milestone summaries

Post-ship writeups for major architecture or feature arcs. One file per milestone.

**Naming:** `YYYY-MM-DD-<short-slug>.md`

**When to add one:** after a multi-PR or multi-phase effort lands on `master` — branches merged, smoke-test done, enough context to be useful six months later.

**Contents (suggested):** overview, architecture diagram, PR/branch table, on-disk layout, install/smoke-test checklist, explicit deferrals.
